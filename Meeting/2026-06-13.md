## 핵심 배경 (코드 기준 확인됨)
- **커미션 레이트가 3군데에 존재 — 의도된 설계, 헷갈림 정리:**
  - `Account.Commission_Rate__c` (Agent 레코드타입/Company) = 그 에이전트의 **기본/대표 레이트**. 여기서 바꾸면 연결된 모든 Insurance로 전파됨. [Account/fields/Commission_Rate__c](../force-app/main/default/objects/Account/fields/Commission_Rate__c.field-meta.xml)
  - `Insurance__c.Commission_Rate__c` = 그 **정책별 sub‑agent 레이트**(free 입력). policy마다 다를 수 있어 agent 기본값을 override. [Insurance__c/fields/Commission_Rate__c](../force-app/main/default/objects/Insurance__c/fields/Commission_Rate__c.field-meta.xml)
  - `Commission_Line__c.Commission_Rate__c` = **formula(읽기전용)**, `Policy_Number__r.Commission_Rate__c`로 Insurance 값을 그대로 가져옴. Commission Line ↔ Insurance는 1‑1. [Commission_Line__c/fields/Commission_Rate__c](../force-app/main/default/objects/Commission_Line__c/fields/Commission_Rate__c.field-meta.xml)
  - → **by‑agent 커미션 리포트는 Insurance 레벨 레이트를 써야 함**(가끔 sub‑agent 레이트가 기본 50%와 다름).
  - Agent는 항상 **K 또는 Evergreen**, **Subagent가 매번 다른 사람** → 실질적으로 **Subagent 레이트만 신경** 쓰면 됨. [Insurance__c/fields/Subagent__c](../force-app/main/default/objects/Insurance__c/fields/Subagent__c.field-meta.xml)
- **Renewal 새 Insurance 생성 흐름:** Quote에서 **Start Sync** + Renewal에서 **Closed Won**(Quote는 Accepted) → Related에 **New Insurance** 자동 생성. [Renewal_Closed_Won_Create_New_Insurance.flow-meta.xml](../force-app/main/default/flows/Renewal_Closed_Won_Create_New_Insurance.flow-meta.xml)
- Renewal의 **Original Premium**은 Original Insurance의 premium을 끌어오는 값(연필 없음=수정 불가). 그래서 Kay가 "갱신 premium을 손으로 못 넣는" 상황 발생.

  - [x] **템플릿 시그니처 제거 완료** ✅ **PROD 처리 2026‑06‑13** — 커스텀 템플릿 17개에서 끝의 `K2 Insurance / K2 Insurance 드림 / K2 보험 드림` 브랜드 줄 + K2 Base 3개의 옛 로고(`$Resource.K2Logo`)+연락처 테이블 제거. `감사합니다 / Thank you` 클로징은 유지. (Lightning=SFX 템플릿은 데이터 레코드라 REST PATCH로 직접 업데이트; 메타데이터 아님.) "ACA 상담 #1"은 이미 푸터가 없어 변경 없음.

- [x] **A3a. "[Not Provided]" 계정 9개 정리 완료** ✅ **PROD 처리 2026‑06‑13**.
  Lead form의 **Company Name이 required**라 개인 리드가 convert 시 `[Not Provided]` 이름의 Account로 생성됨. 단, 이 9개는 **junk가 아니라 실제 고객**(각 1명 Contact = Active 보험 Policy Holder)이었음.
  - 삭제가 막힌 진짜 원인: `Insurance__c.Policy_Holder__c`(**Restrict** lookup → Contact)라 계정 cascade 삭제 시 policy holder Contact가 막음. **정션(ContactInsuranceAssociation)이 아님.**
  - K2 컨벤션 = 개인 고객은 **account 없는 private Contact**(이미 ~664명). 그래서 처리: **Contact 9명의 AccountId 공백 → 빈 계정 9개 삭제.** 보험·Policy Holder 링크 전부 보존 확인.
- [x] **A3b. convert 로직 수정(재발 방지)** ✅ **PROD 배포·활성화 2026‑06‑13 (flow v12)**.
  [Lead_to_Contact_Conversion.flow-meta.xml](../force-app/main/default/flows/Lead_to_Contact_Conversion.flow-meta.xml) "Has valid Company Information?" 결정에 placeholder 체크 추가: `varIsPlaceholderCompany`(`[Not Provided]`/`Not Provided`)가 true면 Invalid 경로로 → Account 안 만들고 **account 없는 Contact**로 변환. (단일문자 `0`/`O`는 기존 `LEN>=2` 조건으로 이미 제외됨.)

- [x] **A4. Health Lead 레이아웃 — Details 먼저.** ✅ 회의 중 변경(몇 분 뒤 적용).
